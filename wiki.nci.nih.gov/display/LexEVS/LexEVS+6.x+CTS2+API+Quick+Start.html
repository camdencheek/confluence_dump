CTS2 Client Options

REST calls are not platform specific. They can be integrated into a wide variety of platforms.
  • Browser
  • Java
  • Scala
  • Python
  • JavaScript
  • Unix Command line
  • Any platform able to handle REST calls and XML or JSON formatted responses

CTS2 Framework Serialization snippet
A reference implementation framework in Java includes parsing and marshaling utilities as well as CTS2 model elements in Java to serialize the JSON responses. CTS2 Bulk Term Download Extension

LexEVS 6.1 features a bulk download REST interface that is an extension to the LexEVS CTS2 service. It provides users with the ability to download entire term sets from a given coding scheme or map to a text file

Example
https://lexevscts2.nci.nih.gov/lexevscts2/exporter/codingscheme?codingschemes=GO

Map Bulk Download

Example
https://lexevscts2.nci.nih.gov/lexevscts2/exporter/map?map=NCIt_to_ChEBI_Mapping

Parameters for Rest Call

URL:
  • /lexevscts2/exporter/codingscheme or
  • /lexevscts2/exporter/map
Parameters:codingschemes or map - (Optional) CodingSchemes or map to include (comma-separated). Default: allfields - (Optional) Content fields to output.
