If your user account has the group administrator or system administrator role, you can view a DOC report.
  1. Log in as described in Logging In via the GUI . The Dashboard appears.
  2. Click the Reports tab. The Reports page appears.
  3. From the Report Type list, select DOC Report.
  4. In the DOC field, type a value. The system displays only the results that contain the specified value. Select a DOC.
  5. (Optional) Adjust the date range in the From Date and To Date fields.
  6. (Optional) Specify whether to Include Archive Summary in the report.
  7. Click Generate. The report appears.
The following table describes the data available in the report:
Data Description
Total Registered Users The total number of users assigned to the selected DOC.
Total Archived Data Size (Bytes) The total size of the archived data in the selected DOC:
  • The first number is in bytes.
  • The second number (in parentheses) is in SI units.
Archive Summary A summary of the data. Each line provides the storage provider (vault), name of the storage bucket, and the size of the data within it.
Largest Archived Data File Size (Bytes) The size of the largest archived data file in the selected DOC:
  • The first number is in bytes.
  • The second number (in parentheses) is in SI units.
Average Archived File Size (Bytes) The average size of an archived data file in the selected DOC:
  • The first number is in bytes.
  • The second number (in parentheses) is in SI units.
Total Number of Data Objects Archived The total number of archived data files in the selected DOC.
Total Number of Collections The total number of collections in the selected DOC, subdivided by depth in the hierarchy. For example, the FNLCR DOC might have 24 collections at the PI_Lab level, 305 collections at the Project level, 301 collections at the Flowcell level, and so on.
Files <with specified size> The total number of archived data files in the selected DOC, subdivided by size.
