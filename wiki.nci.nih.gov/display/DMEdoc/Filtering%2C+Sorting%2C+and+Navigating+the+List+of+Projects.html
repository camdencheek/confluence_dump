If the data_generator_userid attribute of a PI_Lab collection specifies your user name, the system periodically prompts you by email to review your projects. The Review page and the Update Project dialog box allow you to update the project status, publications, deposition, and retention period, as described in Reviewing a Project . The following table describes how to filter, sort, or navigate in the Review page.
To do this ... Do this ...
Select the number of projects per page In the Show ... entries box, select a number. The minimum number of projects per page is 10 and the maximum is 100.
Filter the list In the Filter box, type a value. The system searches all rows and columns in the table and displays only the rows that contain the specified value. You can also click Display Completed Projects.
Review one or more projects In the first column, click the row for the project you want to review. A check mark appears in that column for that row. You can select multiple projects, but the system will apply the same values to all of the selected projects. To select all rows in the list, click  Select All. For details, refer to Reviewing a Project .
Sort the list Click any column header. To reverse the sort, click it again.
View details about a project For projects that have a description, you can view that description by hovering over the project title. For any project, to view metadata of the collection, click the link in the Path column. The Collection page appears, in a new browser tab. For details, refer to Viewing Metadata via the GUI .
Move to the first page in the list Click First.
Move to the previous page in the list Click Previous.
Move to a specific page in the list Click the page number.
Move to the next page in the list Click Next.
Move to the last page in the list Click Last.
