Most preparations for accessibility should be done in the source files. The following pages explain how to prepare documents for accessibility.
  • Section 508 Compliance for Confluence Wiki Pages
  • Section 508 Compliance Checklists for Various File Types
A few key actions can make content significantly more accessible. Key Points For accessibility:
  • You must not merge cells in tables. Merged table cells are not Section 508-compliant.
  • You must enter meaningful information in each table cell, as blank cells are not Section 508-compliant. Repeat data in each cell to which it applies. Indicate "none" or "not applicable" where appropriate.
  • You must include alt text for every image posted on the wiki. In addition, describe the important aspects of the image in the text. Refer to Adding Alt Text to an Image .
Refer also to Accessibility and 508 Compliance (overview) .
