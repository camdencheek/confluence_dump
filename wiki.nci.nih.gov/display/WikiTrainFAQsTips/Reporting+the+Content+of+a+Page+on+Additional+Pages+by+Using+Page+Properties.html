The Page Properties and Page Properties Report macros provide the capability of showing information on different pages, each from a different point of view. As an example, an "events" report follows for multiple projects. The same report is shown in child pages, one for each project, Report of Project A Events and Report of Project B Events .To create a report like the "events" report that follows
  • Set up the wiki page, with a heading for each project. This can be further refined, for example, with a page for each week and a heading for each project.
  • Set up a Page Properties macro as the content of each section.
    • Type {page prop and when the macro browser opens, select the Page Properties macro.
    • In the text box of the macro, enter a table with the report information. The information must be in a table.
    • In the macro dialog box, enter a Page Properties ID. This is optional but needed for reporting when there is more than one Page Properties macro on a wiki page.
  • Add a label to the page to signify the content. This is needed for reporting.
To create a page that reports the events for one project, like Report of Project A Events and Report of Project B Events
  • Create the wiki page for the project report.
  • Enter a Page Properties Report macro on the page for each set of events you want to report for that project.
    • Type {page prop and when the macro browser opens select Page Properties Report.
    • In the macro dialog box, enter the label of the page with the information to report; the Page Properties ID if there is more than one Page Properties macro; and other parameters as needed. In the examples, the "Columns to show" are specified in the order they should appear.
