If you want two wiki pages to display the same content, consider linking from one page to another, or using an Include page, rather than copying the content from one page to another. For information on using an Include page, refer to Including the Content of a Page on Another Page . Copying may be acceptable when preparing drafts for review before release, or when a new page needs similar content (not the same content) as an existing page.You can copy a single page or a page family.
  • To copy markup from a page, choose one of the following options:
    • Open the page for editing, as described in Editing a Page .
    • Click the Tools (...) icon above the page title, on the right side of the page. Select View Source.
  • To copy a single page, refer to the following page: https://confluence.atlassian.com/conf613/copy-a-page-964960391.html
  • To copy a whole page family, or to selectively copy multiple pages, refer to the following page: https://artemis.atlassian.net/wiki/spaces/SUP/pages/410976363/Copy+Page+Tree+Server+and+Data+Center+-+User+Guide#CopyPageTreeServerandDataCenter-UserGuide-how_to_copy_a_page_tree
When copying a single page, keep in mind the following known issues:
  • https://jira.atlassian.com/browse/CONFSERVER-43208
  • https://jira.atlassian.com/browse/CONFSERVER-55493
To make each copied page useful, you must do the following:
  • Change the page title to something more meaningful than "Copy of <original page name>".
  • Ensure all the links from the copied page open the intended targets. For example, if you are copying a family of pages, edit the links so they open pages with the new page titles, rather than the original pages.
