1. Search for the data of interest, as described in Finding Data .
  2. In the search results, click the link for the asset of interest. The Asset Details page appears with the following lists:
    • The asset link. To copy this link, click the copy icon. The system copies the URL to your clipboard. You can send this URL to users who want to download asset files.
    • The program, study, and asset names.
    • The asset path. This path is useful to API programmers.
    • A list of asset attributes and attribute values.
      • If an attribute value begins with http or https, MoDaC displays it as a hyperlink.
      • In a reference dataset, the value in the Applicable Model field is a comma-separated list.
    • On a desktop, a list of files. If you have generated predictions in this asset, or if you belong to an access group associated with the predictions in this asset, a Predictions tab lists those prediction files.
The following two tables describe the actions you can take in the list of files.You can perform the following the actions in the list of files without logging in:
To do this ... Do this ...
Select the number of files per page In the Rows per Page box, select a number. The minimum number of files per page is 10 and the maximum is 100.
Sort by name or file size Click the column header.
View all user metadata and key system metadata for a file or folder Click the Metadata icon ( ) next to the filename or folder name. (For recently-updated metadata, click this icon one hour after the update.) If an attribute value begins with http or https, MoDaC displays it as a hyperlink.
Move to the previous page of results Click the previous icon.
Move to the next page of results Click the next icon.
If you uploaded the asset, or if you belong to an access group associated with the asset, you can perform the following the actions in the list of files after logging in. (For instructions, refer to Logging In .)
To do this ... Do this ...
Generate predictions (if available) Click Generate Predictions. For details, refer to Generating Predictions .
Perform model analysis (if available) Click Perform Model Analysis. For details, refer to Performing Model Analysis Using a Reference Dataset .
Filter the list by filename or partial filename
  1. If the asset has any subcollections, expand the ones in which you want to filter files.
  2. In the Filter field, start typing all or part of a filename.
Copy the path for a file or folder In the row for the file or folder of interest, click the Copy Path icon ( ). The system copies the path to your clipboard.
Edit the metadata for a file or folder In the row for the file or folder of interest, click the Edit Metadata icon ( ). For details, refer to Editing Metadata .
Download the metadata for a file In the row for the file of interest, click the Download File Metadata icon ( ). A message appears. For details, refer to Downloading Metadata for an Asset .
Download the metadata for multiple files Select the Batch Select checkbox. If the asset has subfolders, the system selects only the files at the top level. (To exclude a file from download, click the checkbox for that row.) Click Download Selected Metadata. A message appears. For details, refer to Downloading Metadata for an Asset .
Download a file or folder In the row for the file or folder of interest, click the Download icon ( ). The Download page appears. For details, refer to Downloading Data .
Download multiple files Select the Batch Select checkbox. If the asset has subfolders, the system selects only the files at the top level. (To exclude a file from download, click the checkbox for that row.) Click Download Selected Files. The Download page appears. For details, refer to Downloading Data .
Delete a file In the row for the file of interest, click the Delete File icon ( ). A confirmation message appears. For details, refer to Deleting Files . If the following criteria are met, you can delete a file:
  • You own the data. That is, you created the file.
  • The parent collection is not public. For information, refer to Sharing Collections with Groups .
  • The data file is less than 90 days old.
For actions you can take in the list of prediction files, refer to Exploring Predictions in an Asset .
