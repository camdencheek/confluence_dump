If the following criteria are all true, you can upload one or more files from Google Cloud to an asset in MoDaC:
  • You have a MoDaC user account with upload permission.
  • You are member of a group that has permission to edit that asset (or you created that asset).
When you upload data, you become the owner of that data. (Ownership cannot be changed.)
  1. Log in, as described in Logging In .
  2. Click Upload in the header. The upload page appears.
  3. Navigate to the asset:
    1. Select an existing program or register a new one.
    2. Select an existing study or register a new one.
    3. Select an existing asset or register a new one.
    For instructions, refer to Adding a Collection .
  4. You have the following choices:
    • To upload to the asset, click Upload Files to Asset.
    • To upload to a subcollection, navigate to that subcollection and click the upload icon next to it.
    The Upload Files page appears.
  5. Select Transfer from Google Cloud. Click Authorize MoDaC to Access Your Google Cloud. If a Google page appears, follow the prompts. The MoDaC upload page reappears, indicating the successful generation of an access token.
  6. Specify the following information:
    • The name of the source Google Cloud bucket.
    • The path to and the name of the file in the source bucket.
  7. Click Upload. The system transfers the data based on your selections.
  8. When the system displays the task ID, consider clicking that link to view the progress of the upload. For instructions, refer to Viewing Status .
Google periodically invites users to strengthen the security of their Google accounts. If you follow the Google prompts to remove MoDaC, you can authorize access again the next time you try to upload data from Google Cloud in MoDaC.
