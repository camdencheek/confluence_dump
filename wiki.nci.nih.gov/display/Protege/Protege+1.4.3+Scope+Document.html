The purpose of this document is to collect, analyze, and define high-level needs and features of the National Cancer Institute Center for Biomedical Informatics and Information Technology (NCI CBIIT) Protégé Release 1.4.3. This document focuses on the functionalities proposed by the product stakeholders and target users in order to make it a better product. The use-case and supplementary specifications document will detail how the framework will fulfill these needs.

Vision and Needs

The objective of this release of Protégé 1.4.3 is to provide bugs fixes found in 1.4.2 Protégé release but was not considered a showstopper for the 1.4.2 release.

Current Solution

Protégé 1.4.2 is currently in production.

Technical Environment

This product uses the following technical components which have been derived from the current NCICB Technology Stack .
Component Description
Client Application The Protégé client is java based and provides a complete editing environment for Protégé databases, call ontologies. It uses the same plugin architecture to support extensions, called Tabs. For example, the NCI Edit Tab, Workflow, Changes, and Prompt, are examples of plugin tabs. Java 1.5.x
Application Server None
Database Server MySQL
Operating System Any that supports Java: Windows XP, Linux, OS X
[caCORE:Other] Portable ANSI-SQL compliant relational schemas
