National Cancer Institute Center for Bioinformatics Protege 1.4.1 Patch Release Notes 26 February 2010

Product Changes

The main objective of this patch release of Protégé 1.4.1 is to provide fixes to performance bugs found after the initial release of NCI Protégé 1.4.1 to production. This patch release maintains acceptable performance levels when the size of the Change Ontology increases to unusually high levels (e.g. after multiple batch loads and batch edits).

New Feature Highlights

No additional features were included as part of this release.

Behavior to be Aware Of

GF22902 - You are able to add roles and individuals even though they are pre-merged. You can still edit premerged concepts. GF26478 - Cannot add role after editing a parent due to loss of focus in class browser WORKAROUND: When you re-select the concept in the "Class Browser" the "Create a Role" dialog will appear as intended and with no error. This is a Workaround for this issue.
