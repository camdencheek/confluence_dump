The purpose of this document is to collect, analyze, and define high-level needs and features of the National Cancer Institute Center for Biomedical Informatics and Information Technology (NCI CBIIT) Protégé Release 1.4.1. This document focuses on the functionalities proposed by the product stakeholders and target users in order to make it a better product. The use-case and supplementary specifications document will detail how the framework will fulfill these needs.

Vision and Needs

The objective of this release of Protégé 1.4.1 is to provide bugs fixes found in previous Protégé releases as well as completing feature request for formatting of the complex properties and improving performance for batch edits. This release will be performed over one main development iteration and a second iteration to cover any items found in UAT: Iteration B0 - Development of Scoped Features and Bug Fixes Iteration B1 - Bug fixes found from user acceptance testing

Current Solution

Protege 1.4 is currently in production.

Technical Environment

This product uses the following technical components which have been derived from the current NCICB Technology Stack .
Component Description
Client Application The Protege client is java based and provides a complete editing environment for Protege databases, call ontologies. It uses the same plugin architecture to support extensions, called Tabs. For example, the NCI Edit Tab, Workflow, Changes, and Prompt, are examples of plugin tabs. Java 1.5.x
Application Server None
Database Server MySQL
Operating System Any that supports Java: Windows XP, Linux, OS X
[caCORE:Other] Portable ANSI-SQL compliant relational schemas
