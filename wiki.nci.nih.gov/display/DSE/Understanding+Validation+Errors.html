The following table lists validation errors that may appear and offers possible solutions:
Validation Error Solution
No value column (no tab character). Check whether the information in the file you are importing is tab-delimited.
Unable to parse investigation file. Is this file a valid ISA TAB investigation? Check whether you are importing an investigation file, rather than a study or assay file.
Metadata files not found Check whether your metadata files are in the same folder as your investigation file. That is where the system tries to find them.
Investigation File Missing Required Headers
  • If you have imported an empty investigation file, consider using the Metadata Utility to create a new investigation instead. For instructions, refer to Creating an Investigation .
  • Otherwise, for each specified header, add an element group, as described in Adding an Element or Element Group .
Investigation File missing fields required by the DCC Click each link in the Validation panel. For each required field, add an element, as described in Adding an Element or Element Group .
Study processes must be defined and have steps. For each study, add one or more processes with steps. For instructions, refer to Adding a Process and Adding a Process Step .
Study processes must begin with a Source. At the start of each process, add a source step. For instructions, refer to Adding a Process Step .
Study processes must end with a Sample. At the end of each process, add a sample step. For instructions, refer to Adding a Process Step .
Assay processes must be defined and have steps. For each assay, add one or more processes with steps. For instructions, refer to Adding a Process and Adding a Process Step .
Assay processes must begin with a Sample. At the start of each process, add a sample step. For instructions, refer to Adding a Process Step .
Sample name is required on Sample nodes. For each sample process step, specify a sample name. For instructions, refer to Editing One or More Process Steps .
